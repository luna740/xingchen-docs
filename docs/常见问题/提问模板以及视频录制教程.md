# 提问模板以及视频录制教程

分类：常见问题
更新时间：2026-05-20T22:13:32+08:00

**本文说明如何通过自助提交模块反馈问题，以及如何补充截图和录屏，方便问题更快被记录、定位和处理。**

> 注意：星辰需要能复现问题后才能准确处理。请按 `/q` 或 `/qx` 格式提交，普通聊天不会触发机器人记录。

## 一、自助提交模块说明

为了更好、更快响应问题，反馈时请使用自助提交模块。机器人会根据命令格式记录问题，并对缺少关键信息的提交进行提示。

1. 涉及 WhatsApp 号码的问题，使用 `/q` 提交。
2. 涉及坐席账号或管理员账号的问题，使用 `/qx` 提交。
3. `/q` 和 `/qx` 必须放在消息开头。
4. 普通聊天不会触发机器人记录。

## 二、号码问题提交格式

涉及 WhatsApp 号码的问题，请使用以下格式：

```text
/q 号码 截图 问题描述
```

提交要求：

1. `/q` 不可缺少，缺少后机器人不会记录为问题。
2. `号码` 不可缺少，缺少号码时机器人会提示补充号码。
3. `截图` 需要能判断问题发生的位置，例如是哪个私聊对话、哪个群、哪个账号页面。
4. `问题描述` 需要说明异常现象和期望结果。
5. 如果截图内容无法判断问题发生位置，机器人会提示补充更完整的截图。

示例：

```text
/q 86135****3252 截图 发送消息失败，页面提示发送异常
```

> 提示：涉及少于 5 个号码时，建议列出全部号码；涉及超过 5 个号码时，至少列出 5 个典型号码，并附对应截图。

## 三、坐席或管理员问题提交格式

涉及坐席账号或管理员账号的问题，请使用以下格式：

```text
/qx 坐席账号或者管理员账号 截图 问题描述
```

提交要求：

1. `/qx` 必须放在消息开头。
2. `/qx` 命令不审核内容，请自行确认账号、截图和问题描述是否完整。
3. 截图需要能看出发生问题的页面位置。
4. 问题描述需要说明具体操作和异常表现。

示例：

```text
/qx admin001 截图 坐席列表打开后无法新增坐席
```

## 四、截图和录屏要求

1. 截图请保留网址下方的完整页面区域，不能只截局部按钮或局部提示。
2. 录屏请从进入问题页面前开始，直到异常提示出现为止。
3. 如果问题可以稳定复现，请优先提供录屏。
4. 截图或录屏中需要能判断问题发生的账号、对话、群或页面。

## 五、使用 PowerPoint 录屏

PowerPoint 2013 及以上版本可以直接录制屏幕。

1. 打开 PowerPoint。
2. 点击顶部【插入】选项卡。
3. 点击右侧【屏幕录制】。
4. 点击【选择区域】，拖动选择需要录制的范围。
5. 如需全屏录制，可按 `Windows + Shift + F`。
6. 点击红色【录制】按钮，倒计时结束后开始录制。
7. 操作完成后点击【停止】。
8. 视频会自动插入到 PPT 页面中。
9. 右键点击视频，选择【将媒体另存为】，保存为 `.mp4` 文件。

## 六、使用 LICEcap 录屏

LICEcap 适合录制较短的操作过程。

1. 安装并打开 LICEcap。
2. 调整透明录制框，让它覆盖需要录制的页面区域。
3. 点击右下角【Record】。
4. 选择保存位置和文件名。
5. 开始录制后，按正常流程复现问题。
6. 需要暂停时按 `Ctrl + Alt + P`。
7. 录制完成后点击【Stop】保存文件。

> 提示：录屏文件较大时，请先确认能正常打开播放，再提交给处理人员。
